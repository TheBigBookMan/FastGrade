# End-to-End Testing Plan

## Purpose
Validate critical user flows from a user perspective.

## Covered Flows
- User signup/login
- Core feature execution
- Payment or irreversible actions

## Not Covered
- Admin-only workflows
- Edge-case permutations
- Visual styling

## Characteristics
- Minimal number of tests
- High business value
- Slower, but trusted

## Tooling
- <E2E framework>
