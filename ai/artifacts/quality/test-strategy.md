# Test Strategy

## Purpose
Define the overall testing approach and quality expectations.

## Testing Goals
- Prevent regressions
- Validate business rules
- Enable safe refactoring

## Test Levels
- Unit tests
- Integration tests
- End-to-end tests

## In Scope
- Business logic
- API behavior
- Critical user flows

## Out of Scope
- Visual pixel perfection
- Third-party service internals
- Load testing (unless specified)

## Quality Bar
- All acceptance criteria must be covered
- Tests must be deterministic
- Flaky tests are treated as failures
