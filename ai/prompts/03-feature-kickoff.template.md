## Acting As
Product Agent (supported by Staff Engineer)

## Mandatory Reads
- /ai/constitution.md
- /ai/artifacts/product/README.md

## Inputs
- Feature request text
- Constraints (deadline, compliance, systems touched)

## Tasks
1. Create:
   - /ai/artifacts/product/feature/<feature_slug>/README.md
   - /ai/artifacts/acceptance/feature/<feature_slug>.md
2. Feature brief must define:
   - problem statement
   - user roles
   - in scope / out of scope
   - assumptions
   - risks and open questions
3. Acceptance criteria must define:
   - happy paths
   - validation rules
   - error cases
   - edge cases
   - non-functional constraints

## Constraints
- Do not define implementation details
- Do not override technical standards

## Output
- Summary of artifacts created
- Assumptions list
- Stop
